## 1. 아키텍처 개요

### 1.1 시스템 구성

```
┌─────────────────────────────────────────────────────────────────┐
│                    proposal-agent v5.1                           │
│                                                                 │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐       │
│  │  RFP     │→│ Content  │→│ Design   │→│ slide_kit│→ PPTX  │
│  │ Analyzer │  │Generator │  │  Agent   │  │  v3.8   │        │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘       │
│       ↑              ↑             ↑                            │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐                     │
│  │Think Tank│  │   IP     │  │  Gamma   │                     │
│  │   DB     │  │Researcher│  │   MCP    │                     │
│  └──────────┘  └──────────┘  └──────────┘                     │
│                                    ↓                            │
│  ┌──────────┐              ┌──────────────┐                    │
│  │  Image   │              │GammaMCPBridge│                    │
│  │ Pipeline │              │    v7.1      │                    │
│  └──────────┘              └──────────────┘                    │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 3-Layer 분리 아키텍처

| Layer | 모듈 | 역할 |
|-------|------|------|
| **콘텐츠** | slide_kit v3.8 | PPTX 생성 엔진 (40+ 컴포넌트, 20 레이아웃) |
| **디자인** | DesignAgent + Gamma MCP | 컬러/폰트/레이아웃/이미지 스타일 결정 |
| **톤** | ContentTone | 감성 레벨, IP 깊이, 프로그램 네이밍, Win Theme 스타일 |

### 1.3 데이터 플로우

```
RFP (PDF) → RFPAnalysis
    ↓
Think Tank 조회 → DesignBrief (레퍼런스 기반)
    ↓
Gamma get_themes() → ThemeRecommendation[] (선택적)
    ↓
DesignAgent.merge() → MergedDesignBrief
    ↓
register_theme() + apply_theme() → slide_kit 컬러 변경
    ↓
생성 스크립트 (generate_*.py) → PPTX 초안
    ↓
ImagePipeline → IMG_PH 교체 (선택적)
    ↓
GammaMCPBridge → Gamma 디자인 워싱 (선택적)
```

### 2.2 DesignAgent (디자인 결정 에이전트)

**위치**: `src/agents/design_agent.py`

싱크탱크 레퍼런스(1순위)와 Gamma 추천(2순위)을 병합하여 최적 디자인 결정을 내리는 에이전트.

**우선순위**: `사용자 직접 지정 > 사용자 편집 오버라이드 > 싱크탱크 레퍼런스 > Gamma 추천 > 기본값`

| 메서드 | 입력 | 출력 |
|--------|------|------|
| `get_think_tank_brief()` | project_type, industry | DesignBrief (dict) |
| `interpret_gamma_themes()` | gamma_themes_data, keywords | ThemeRecommendation[] (상위 3개) |
| `merge()` | tt_brief, gamma_recs, custom_colors | **MergedDesignBrief** |
| `generate_full_brief()` | 모든 파라미터 통합 | MergedDesignBrief (원스텝) |
| `register_to_slide_kit()` | MergedDesignBrief | slide_kit 테마 등록 완료 |

**병합 로직**:
```
1. 싱크탱크 컬러 vs Gamma 테마 컬러 → 프로젝트에 더 적합한 쪽 선택
2. 싱크탱크에 컬러 데이터가 있으면 우선, 없으면 Gamma 추천 사용
3. 레이아웃/배경/컴포넌트는 항상 싱크탱크 기반 (Gamma는 이 데이터 없음)
4. 이미지 스타일만 Gamma 추천 적극 반영
5. 콘텐츠 톤은 싱크탱크 ContentToneProfile 기반
```

### 2.3 MergedDesignBrief (디자인 스키마)

**위치**: `src/schemas/design_schema.py`

전체 디자인 결정의 최종 산출물. 생성 스크립트, 이미지 파이프라인, Gamma 연동 등 모든 하위 모듈이 참조.

```python
@dataclass
class MergedDesignBrief:
    # 프로젝트 메타
    project_name: str
    project_type: str
    industry: str

    # 싱크탱크 기반
    section_weights: Dict[str, int]           # Phase별 슬라이드 수
    component_targets: Dict[str, int]         # 컴포넌트별 사용 목표
    layout_distribution: Dict[str, float]     # 레이아웃 유형별 빈도
    background_schedule: List[str]            # 슬라이드별 배경색
    visual_density_targets: Dict[str, float]  # 이미지/다이어그램/텍스트 비율
    content_patterns: List[Dict]
    program_templates: List[Dict]

    # 콘텐츠 톤 (v5.1)
    content_tone: ContentToneBrief            # 감성 레벨, IP 깊이, 톤 규칙

    # Gamma 보완
    theme_name: str
    colors: Dict[str, Tuple[int,int,int]]     # slide_kit C[] 매핑
    font_weight_primary: str
    image_style: ImageStyleGuide
    gamma_recommendations: List[ThemeRecommendation]

    # 메타
    source_references: List[str]
    confidence: float
```

**ContentToneBrief** (v5.1 신규):
```python
@dataclass
class ContentToneBrief:
    emotional_tone_level: int      # 1(사무적) ~ 5(풀 내러티브)
    narrative_framing_style: str   # worldview_based / data_driven / emotion_led / hybrid
    core_metaphor: str             # "니케의 지상 귀환" 등
    ip_depth_score: float          # 0.0 ~ 1.0
    ip_lore_terms: List[str]       # IP 세계관 용어
    program_naming_style: str      # ip_narrative / functional / branded / hybrid
    win_theme_style: str           # ip_worldview / keyword_functional / emotional_hook
    tone_rules: List[str]          # 생성 스크립트 직접 참조 규칙
```

### 2.4 slide_kit v3.8 (콘텐츠 엔진)

**위치**: `src/generators/slide_kit.py`

40+ 컴포넌트, 20 레이아웃, 5+N 테마를 제공하는 경량 PPTX 생성 엔진.

**핵심 카테고리**:
| 카테고리 | 컴포넌트 수 | 대표 함수 |
|---------|-----------|----------|
| 상수 | 10+ | `C`, `SW`, `SH`, `ML`, `CW`, `SZ`, `FONT`, `FONT_W`, `SHADOW`, `GRAD` |
| 도형 | 6 | `R()`, `BOX()`, `OBOX()`, `RBOX()`, `ORBOX()`, `CARD()` |
| 텍스트 | 3 | `T()`, `RT()`, `MT()` |
| 도식화 | 15+ | `FLOW()`, `COLS()`, `PYRAMID()`, `TABLE()`, `KPIS()`, `GANTT_CHART()` 등 |
| 차트 | 3 | `BAR_CHART()`, `PIE_CHART()`, `LINE_CHART()` |
| 슬라이드 | 6 | `slide_cover()`, `slide_section_divider()`, `slide_toc()` 등 |
| 테마 | 4 | `register_theme()`, `apply_theme()`, `reset_theme()`, `list_themes()` |

**v3.8 핵심 개선**:
- `register_theme()` — 외부 테마 동적 등록 (DesignAgent 연동)
- `apply_theme()` — 파생 컬러 자동 재계산 (primary_dark, secondary_light 등 21색)
- `_ensure_emu()` — EMU int/float 혼합 안전 처리
- `VStack` — 자동 Y좌표 계산, 겹침 방지
- COLS/SPLIT_VISUAL 자동 클램핑 — 슬라이드 경계(7.0") 초과 방지

### 2.5 GammaMCPBridge v7.1 (Gamma 연동)

**위치**: `src/integrations/design_bridge.py`

Gamma MCP 도구와 Python 코드 사이의 브릿지. API 폴링 자동 다운로드 지원.

**전체 파이프라인**:
```
1. run_gamma_pipeline(pptx, brief)     → GammaPipelineResult (텍스트 추출 + 파라미터 빌드)
2. Claude Code: Gamma MCP generate()   → {"generationId": "xxx", "status": "pending"}
3. handle_gamma_response(response)     → 3-way 다운로드 전략 결정
4. poll_and_download_gamma(gen_id)     → PPTX 자동 다운로드 (API 폴링)
```

**3-way 다운로드 전략**:
| 전략 | 조건 | 동작 |
|------|------|------|
| `direct` | 응답에 exportUrl 포함 | 즉시 HTTP 다운로드 |
| `api_poll` | GAMMA_API_KEY 환경변수 존재 | 5초 간격 폴링 → downloadLink 획득 → 다운로드 |
| `browser` | API 키 없음 (fallback) | Claude Code가 브라우저로 수동 내보내기 |

**API 기술 상세**:
- **Gamma Public API v1.0**: `https://public-api.gamma.app/v1.0/generations`
- **인증**: `X-API-KEY: sk-gamma-xxxxx`
- **Cloudflare 대응**: `User-Agent: GammaBridge/1.0` 필수 (Python urllib 기본값 차단)
- **연속 에러 보호**: `max_consecutive_errors=10`
- **다운로드 타임아웃**: 600초 (대용량 PPTX 대응, 검증: 319MB 파일 성공)

**사용자 편집 역파싱** (14개 카테고리):
```
지오메트리, 채우기, 선/테두리, 타이포그래피, 단락 서식,
그림자/효과, 이미지 교체, 도형 유형 변경, 도형 추가/삭제,
슬라이드 순서, 배경, 노트, 마스터/레이아웃, 기타
```

## 3. 워크플로우 (6-Step)

### STEP 1: RFP 분석
```
입력: 제안요청서/테스트 XX/*.pdf
출력: RFPAnalysis (프로젝트명, 발주처, 과업 범위, 평가 기준, 예산, 일정)
모듈: src/agents/rfp_analyzer.py
```

### STEP 2: 콘텐츠 기획
```
입력: RFPAnalysis
출력: Impact-8 Phase 구조 (Phase 0~7), Win Theme 3개, Action Title, KPI+산출근거
모듈: src/agents/content_generator.py
```

### STEP 2.5: 디자인 에이전트 (5단계)
```
2.5a: 싱크탱크 조회 → DesignBrief (레퍼런스 기반)
2.5b: Gamma 테마 조회 → ThemeRecommendation[] (선택적)
2.5c: 병합 → MergedDesignBrief (컬러, 배경, 컴포넌트, 이미지 스타일)
2.5d: 콘텐츠 톤 적용 → ContentToneBrief (감성 레벨, IP 깊이, 톤 규칙)
2.5e: 사용자 편집 역파싱 → 디자인 오버라이드 (이전 버전 반영)
```

### STEP 3: 생성 스크립트 작성
```
입력: RFPAnalysis + MergedDesignBrief + ContentToneBrief
출력: output/테스트 XX/generate_제안서.py
규칙: slide_kit import 필수, LAYOUTS 활용, tone_rules 준수
```

### STEP 4: 실행 및 검증
```
입력: generate_제안서.py
출력: output/테스트 XX/제안서.pptx (40~80장)
검증: 겹침/공백/경계 규칙 준수 확인
```

### STEP 5: 이미지 삽입 (선택적)
```
입력: PPTX + MergedDesignBrief
출력: PPTX (IMG_PH → 실제 이미지)
환경변수: PEXELS_API_KEY (필수)
```

### STEP 6: Gamma 디자인 워싱 (선택적)
```
입력: PPTX + MergedDesignBrief
출력: Gamma 디자인 PPTX (output/*/gamma_pptx/)
환경변수: GAMMA_API_KEY (필수)
플로우: run_gamma_pipeline → generate() → poll_and_download_gamma
```
