Proposal Agent v4.0 — 구현 계획서

Context
레퍼런스(수주 성공작, 93p)와 현재 AI 생성물(39p)의 갭 분석 결과, 근본적인 한계가 식별됨:

IP/브랜드 심층 이해 부재 — RFP 밖의 실제 데이터(MAU, 커뮤니티 밈, 경쟁사 등)를 조사하지 않음
콘텐츠 깊이 부족 — 프로그램을 테이블 1줄로 요약 vs 레퍼런스는 프로그램당 2-3페이지
레퍼런스 학습 체계 부재 — 기존 수주 성공 제안서의 패턴/디자인/구조를 축적하지 못함
시각 자료 부재 — IMG_PH 플레이스홀더만 존재, 실제 이미지 없음
디자인 고도화 경로 없음 — PPTX 생성 후 끝, Gamma 등 연동 불가
사용자 요구사항:

모든 모듈이 ON/OFF 가능하고 유기적으로 연결
Think Tank(싱크탱크)로 레퍼런스 지속 학습 + 중복 방지
디자인 요소(폰트/컬러/레이아웃)도 학습
산업 보편 타당성 (게임뿐 아니라 모든 형태의 제안서)
이미지 서칭/AI 생성 파이프라인
Gamma 디자인 연동 (사용자 확인 후)
Phase 1: 파이프라인 인프라 + ON/OFF 제어 (Foundation)
모든 후속 모듈의 기반. 이것 없이는 매번 orchestrator를 직접 수정해야 함.

신규 생성 파일
config/pipeline_config.yaml — 파이프라인 ON/OFF 설정

yaml
pipeline:
  steps:
    rfp_analysis: { enabled: true }
    ip_research: { enabled: true, web_search: true, community: true }
    think_tank_retrieval: { enabled: true }
    content_generation: { enabled: true }
    image_pipeline: { enabled: false }
    slide_generation: { enabled: true }
    validation: { enabled: true }
    design_bridge: { enabled: false, provider: "gamma" }
  think_tank: { enabled: true, db_path: "think_tank/references.db" }
```

**`src/pipeline/config.py`** — PipelineConfig Pydantic 모델
- YAML 로드 + 환경변수 오버라이드
- 각 step별 StepConfig 모델 (enabled, 개별 옵션)
- `get_pipeline_config()` 싱글톤

**`src/pipeline/engine.py`** — PipelineEngine
- `register_step(name, step_instance)` — 단계 등록
- `execute(context)` — enabled된 단계만 순차 실행
- `PipelineContext` — 단계 간 데이터 전달 객체 (rfp_analysis, ip_research, content 등)

**`src/pipeline/steps.py`** — PipelineStep 추상 클래스
- `execute(context) -> context` 인터페이스
- 각 기존 기능을 Step으로 래핑 (RFPAnalysisStep, ContentGenerationStep 등)

## Phase 2: Think Tank 핵심 (레퍼런스 DB + 디자인 추출)

> 사용자 요청 #1, #2, #3 대응. 레퍼런스 투입 → 구조화 저장 → 검색.

### 신규 생성 파일

**`think_tank/__init__.py`**

**`think_tank/models.py`** — DB 모델
- `ReferenceDocument`: id, file_hash, doc_type, industry, project_type, won_bid, total_pages, structure, design_profile
- `DesignProfile`: colors, fonts, font_hierarchy, layout_patterns, bg_style
- `ContentPattern`: pattern_type, section_context, structure, slide_count
- `ProgramTemplate`: name, mechanism, reward_structure, operation_plan (이벤트 프로그램 패턴)

**`think_tank/ingestion/pdf_ingester.py`**
- PDF → 텍스트 + 이미지 추출 (pypdf + pdfplumber 재사용)
- 섹션 구조 자동 분석

**`think_tank/ingestion/pptx_ingester.py`**
- PPTX → 슬라이드별 텍스트 + 레이아웃 + 디자인 추출
- python-pptx로 색상/폰트/도형 위치 자동 파싱

**`think_tank/ingestion/design_extractor.py`**
- PPTX에서 컬러 팔레트, 폰트 계층, 레이아웃 패턴 자동 추출
- `DesignProfile` 객체 생성

**`think_tank/ingestion/dedup_checker.py`**
- SHA-256 파일 해시로 정확한 중복 차단
- 이미 학습된 파일 재투입 방지

**`think_tank/db.py`** — SQLite 기반 저장소
- `save_reference()`, `get_reference()`, `search_by_type()`, `exists(file_hash)`

**`think_tank/retrieval.py`** — 검색 엔진
- `search_similar(project_type, industry, top_k)` — 유사 레퍼런스 반환
- `get_design_patterns(project_type)` — 해당 유형의 디자인 패턴 반환
- `get_content_patterns(section)` — 섹션별 콘텐츠 패턴 반환
- `get_program_templates(industry)` — 이벤트 프로그램 템플릿 반환

**`src/pipeline/steps.py`에 추가** — `ThinkTankRetrievalStep`
- 파이프라인에서 자동 호출
- PipelineContext에 `similar_references`, `design_patterns`, `content_patterns` 추가
```
