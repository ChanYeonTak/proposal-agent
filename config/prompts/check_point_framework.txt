# CHECK POINT 프레임워크

레퍼런스(수주 성공 제안서)에서 발견된 핵심 설득 구조입니다.
모든 프로그램/전략 섹션에 이 프레임워크를 적용하세요.

## 구조

### ANALYTICS (분석)
- 데이터로 문제/기회 정의
- "왜 이것이 필요한가?"에 대한 근거
- 구체적 수치, 트렌드, 벤치마크

예시:
- "2024 AGF 관람객 중 72%가 게임 체험 부스를 1순위로 꼽았습니다"
- "브라운더스트2 유저의 65%가 오프라인 이벤트 참여 의향"

### SOLUTION (해결책)
- 분석에 기반한 구체적 프로그램/전략 제시
- "이렇게 해결합니다"의 상세 계획
- 메커니즘, 보상 구조, 운영 방법 포함

예시:
- "IP 세계관 기반 4단계 체험 존 구성"
- "보상형 미션 시스템: 스탬프 5개 수집 -> 한정판 굿즈 교환"

### EFFECT (효과)
- 정량적 기대효과 제시
- KPI + 산출 근거
- 성공 사례 참조

예시:
- "예상 참여자: 일 3,000명 (전년 대비 +40%)"
- "SNS 도달: 누적 500만 뷰 (인플루언서 3인 x 평균 100만 뷰 + 자연 발생)"

## 적용 규칙

1. **모든 프로그램**: Analytics -> Solution -> Effect 순서
2. **수치 필수**: 각 Effect에 반드시 산출 근거 포함
3. **연결**: 전체 KPI와 개별 프로그램 Effect가 합산 가능하게
4. **시각화**: Analytics는 차트/데이터, Solution은 플로우/다이어그램, Effect는 METRIC_CARD

## Phase별 적용

| Phase | Analytics | Solution | Effect |
|-------|-----------|----------|--------|
| Phase 2 (INSIGHT) | ★★★ 중심 | - | - |
| Phase 4 (ACTION) | 각 프로그램 도입부 | ★★★ 중심 | 각 프로그램 마무리 |
| Phase 5 (MGMT) | 리스크 분석 | 대응 방안 | 품질 보증 |
| Phase 7 (ROI) | 투자 분석 | 비용 구조 | ★★★ 중심 |
